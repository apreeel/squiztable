# squiztable

Делает PNG 1920×1080 со страницы вида `https://my.squiz.ru/results/<uuid>`,
где центрирована карточка с таблицей результатов с гарантированным отступом
~50 px со всех сторон. Вокруг карточки — прозрачный фон (PNG с alpha),
чтобы изображение чисто ложилось на слайд с любой подложкой.

Скрипт сам выставляет значения штатных слайдеров на странице (размер шрифта,
ширина таблицы) — то, что в UI не сохраняется между сессиями.

## Установка (Tampermonkey)

Основной и рекомендуемый способ — userscript в браузере. Ничего ставить кроме
расширения не надо.

1. Поставь расширение **Tampermonkey** из Chrome Web Store.
2. Открой `squiztable.user.js` в этом репо в raw-виде — Tampermonkey сам
   предложит установить, нажми **Install**. Альтернатива: Dashboard →
   Utilities → Import from URL → ссылка на raw-файл.
3. Зайди на любую страницу `https://my.squiz.ru/results/<uuid>` — в правом
   нижнем углу появится кнопка **«Скачать PNG»**.
4. Клик → через 3-5 секунд в Downloads появится
   `MMDD_<название турнира>_n.png` (1920×1080, готов к вставке на слайд).

После съёмки настройки слайдеров **остаются применёнными** на странице —
чтобы вернуть исходный вид, нажми F5.

### Параметры

Меняются прямо в начале файла userscript-а (Tampermonkey → Edit → блок
`CONFIG` → Save). Основное:

```js
const CONFIG = {
  viewport: { width: 1920, height: 1080 },
  padding: 50,                // мин. отступ от краёв (px)
  sliderFont: 140,            // значение слайдера размера шрифта
  sliderWidth: "auto",        // "auto" — auto-fit под viewport - 2*padding
  fontSizeOverride: "1.4rem", // CSS-инжект поверх слайдера; null = не трогать
  panelBorder: "2px solid rgba(255,255,255,0.22)", // видимая рамка карточки
  background: null,           // фон карточки; null = цвет body со страницы
  buttonLabel: "Скачать PNG",
};
```

`sliderFont` / `sliderWidth` работают с **Radix UI слайдерами** (`role="slider"`
с `aria-valuenow`), как на squiz. Поиск по соседней подписи (рус/eng:
`T` / `Шрифт` / `Размер` / `Font` — шрифт; `Ширина` / `Width` — ширина).
Фолбэк по DOM-порядку: первый слайдер = шрифт, второй = ширина. Управление
ставит фокус на thumb и шлёт `End` / `Home` / `Arrow*`, как реальный
пользователь.

`sliderWidth: "auto"` — после установки шрифта замеряет ширину карточки и
жмёт ArrowLeft, пока та не станет ≤ `1920 - 2*padding` px (либо слайдер не
упрётся в минимум). Гарантирует строго 50 px отступа слева/справа без
последующего масштабирования.

`fontSizeOverride` — CSS-значение `!important` на `[data-shot-target]` и
потомках. Используй, когда штатного максимума слайдера (140 ≈ 1.225rem)
мало — `"1.4rem"`, `"1.6rem"`, `"18px"`. Применяется после слайдера и до
auto-fit ширины, так что ширина подгонится под новый шрифт.

`background` подставляется как заливка под саму карточку (внутри скруглений).
`null` берёт цвет `<body>` со страницы (тёмная тема). Хочешь белую карточку —
ставь `"#ffffff"`. Снаружи карточки в любом случае прозрачно.

## Как это работает

1. Виртуализированные строки таблицы «материализуются» скроллом до низа и
   обратно.
2. Находятся штатные Radix-слайдеры, выставляются значения из `CONFIG` через
   эмуляцию клавиатуры — SPA реагирует как на ручную настройку.
3. От самой большой `<table>` вверх ищется «панель» — ближайший предок с
   фоном / скруглениями / тенью / рамкой. Она помечается атрибутом
   `data-shot-target`.
4. На панель инжектятся стили съёмки: рамка, фон под карточкой,
   `overflow: hidden` (чтобы шапка не торчала за скруглениями).
5. [`modern-screenshot`](https://github.com/qq15725/modern-screenshot)
   рендерит панель в canvas через SVG `<foreignObject>` — браузер сам делает
   layout, поэтому шрифты, скругления, тени — пиксель-в-пиксель.
6. Canvas панели центрируется на прозрачном холсте 1920×1080 с минимум
   50 px отступа. PNG сохраняется через `URL.createObjectURL` + `<a download>`.

## CLI (`shot.js` через Node) — легаси

Старый вариант на Node + Playwright из терминала. Оставлен в репо как
запасной — основной способ это userscript выше.

Установка (один раз, Windows):

1. Node.js LTS с https://nodejs.org/, перезапусти терминал.
2. В папке проекта: `npm install` (через `postinstall` скачается Chromium
   для Playwright, ~150 МБ).

Использование:

```
shot.bat https://my.squiz.ru/results/0e5b893f-81c9-43d5-b104-aaee55bf458f
```

PNG сохраняется в текущую папку под именем `MMDD_<заголовок турнира>_n.png`.
Опции CLI: `--out <file.png>`, `--selector "<css>"`, `--padding 50`. Все
параметры конфигурируются в верхушке `shot.js` (тот же `CONFIG` по идее, что
в userscript-е, плюс `rowPaddingY` / `lineHeight` / `waitTimeout`). PNG в
этой версии — без прозрачного фона: композит делается на цвет `<body>`
страницы.

## Проблемы

- **Кнопка не появилась** — userscript не загрузился. Tampermonkey →
  Dashboard → проверь что скрипт включён и `@match` совпадает с URL.
- **Шапка таблицы выходит за скруглённые углы карточки** — починено в
  0.2.7 (`overflow: hidden` на панели). Обнови userscript.
- **Под карточкой прозрачно вместо тёмного фона** — починено в 0.2.6
  (фон инжектится перед съёмкой). Обнови userscript.
- **Панель шире 1820 px после auto-fit** — слайдер упёрся в минимум.
  Кадр всё равно отрендерится, но карточка будет масштабирована
  `modernScreenshot`'ом до целевой ширины. Снизь `fontSizeOverride`.
- **(CLI) «No table-like element found»** — авто-детект не нашёл `<table>`.
  Передай селектор через `--selector ".my-class"`.
