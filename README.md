# squiztable

Делает PNG 1920×1080 со страницы вида `https://my.squiz.ru/results/<uuid>`,
где центрирована панель с таблицей результатов с гарантированным отступом
~50px со всех сторон.

Скрипт сам устанавливает значения штатных слайдеров на странице (размер шрифта,
ширина таблицы) — то, что в UI не сохраняется между сессиями.

## Установка (Windows, один раз)

1. Поставь Node.js LTS с https://nodejs.org/, перезапусти терминал.
2. В папке проекта:
   ```
   npm install
   ```
   (через `postinstall` автоматически скачается Chromium для Playwright, ~150 МБ).

## Использование

```
shot.bat https://my.squiz.ru/results/0e5b893f-81c9-43d5-b104-aaee55bf458f
```

PNG сохранится в текущую папку под именем
`MMDD_<заголовок турнира>_n.png`, где:
- `MMDD` — сегодняшняя дата (месяц-день).
- `<заголовок>` — текст из `.text-lg.text-foreground` на странице
  (обычно «Squiz Микс #N · дата · место · подзаголовок»).
- `_n` — счётчик 1, 2, 3…, инкрементируется, чтобы не перетирать
  существующие файлы.

Если задать `--out`, имя берётся из аргумента и логика выше пропускается.

Можно перетащить на `shot.bat` ярлык/ссылку — URL придёт как аргумент.

## Опции CLI

```
node shot.js <url>
  --out <file.png>      имя выходного файла (по умолчанию <uuid>.png)
  --selector "<css>"    css-селектор контейнера, если авто-детект не подходит
  --padding 50          мин. отступ от краёв 1920×1080 (px)
```

## Настройки в `shot.js`

В верхней части файла:

```js
const CONFIG = {
  viewport: { width: 1920, height: 1080 },
  padding: 50,                // мин. отступ от краёв (px)
  sliderFont: 140,            // значение для слайдера размера шрифта (60..140 на squiz) — base-line
  sliderWidth: "auto",        // конкретное значение, или "auto" — стрелка влево, пока панель не влезет, или null
  fontSizeOverride: "1.4rem", // CSS-инжект font-size поверх слайдера (слайдер на максимуме = ~1.225rem). null = не трогать
  rowPaddingY: null,          // паддинг ячеек по вертикали: "4px", или "auto" — сжимать пока высота не влезет, или null
  lineHeight: null,           // line-height ячеек, например "1.0"; null = не трогать
  background: null,           // фон композита; null = взять цвет body со страницы
  waitTimeout: 30000,
};
```

`sliderFont` / `sliderWidth` поддерживают **Radix UI слайдеры** (`role="slider"`
с `aria-valuenow`) и нативные `<input type="range">`. Поиск идёт по соседней
подписи (рус/eng — `T`/`Шрифт`/`Размер`/`Font` для шрифта, `Ширина`/`Width`
для ширины). Если по подписи не определилось — фолбэк: первый Radix-слайдер
в DOM = шрифт, второй = ширина (сейчас на squiz это работает). Управление
ставит фокус на thumb и шлёт `End`/`Home`/`Arrow*`, как реальный пользователь.

`sliderWidth: "auto"` — после установки шрифта замеряет ширину панели и
жмёт ArrowLeft, пока панель не станет ≤ `1920 - 2*padding` px (или слайдер
не упрётся в минимум). Это даёт строго 50px отступа слева/справа без
последующего масштабирования картинки в композите.

`fontSizeOverride` — CSS-значение, которое инжектится `!important` на
`[data-shot-target]` и потомков. Используй, когда штатного максимума
слайдера (140 ≈ 1.225rem) недостаточно — например, `"1.4rem"`, `"1.6rem"`,
`"18px"`. Применяется после слайдера и до auto-fit ширины, так что
ширина подгонится уже под новый размер шрифта.

`rowPaddingY: "auto"` сжимает вертикальный паддинг ячеек (12 → 0 px),
пока высота панели не станет ≤ `1080 - 2*padding`. Полезно, когда команд
много (≥ 25): без этого панель пришлось бы сильно уменьшать в композите.

`background = null` берёт фоновый цвет `<body>` со страницы (т.е. сохраняется
тёмная тема). Если хочешь белый фон вокруг панели — поставь `"#ffffff"`.

## Как это работает

1. Открывается headless Chromium 1920×1080.
2. Загружается страница, виртуализированные строки «материализуются»
   (скролл вниз и обратно).
3. Скрипт находит штатные слайдеры (`<input type="range">`) и проставляет
   значения из CONFIG, дёргая `input`/`change` события — SPA реагирует
   как на ручную настройку.
4. От самой большой `<table>` вверх находится «панель» — ближайший предок
   с фоном/скруглениями/тенью. Она помечается как цель.
5. Делается **element-screenshot** только этой панели (Playwright клипит
   по её bbox — никакая попытка править DOM не нужна).
6. Открывается вторая вкладка 1920×1080 с цветом фона страницы; туда
   вставляется PNG панели через `<img>` с `object-fit: contain`,
   `max-width: 1820px`, `max-height: 980px`. Снимается финальный скриншот.

Это даёт: панель в её натуральном виде (фон, скругления, шрифты), точное
центрирование и гарантированный отступ ≥ 50px.

## Проблемы

- **«No table-like element found»** — авто-детект не нашёл `<table>`. Открой
  страницу в Chrome (F12), копируй селектор контейнера и передай через
  `--selector ".my-class"`.
- **Слайдер не нашёлся / не применился** — скрипт ищет `<input type="range">`
  по русским/английским подписям. Если на сайте кастомный slider-компонент
  без `<input>`, в консоль выведется `not found by label` — тогда настрой
  слайдеры руками в браузере (если это работает; пользователь жаловался,
  что не сохраняется), или открой issue с DOM-структурой слайдера.
- **Панель шире 1820 px** — `object-fit: contain` отмасштабирует её при
  композите. Получится резко (PNG → image rendering), но если хочешь
  максимальное качество, выставь `sliderWidth` так, чтобы панель сразу
  была ≤ 1820 px.
- **Запускается долго первый раз** — Playwright качает Chromium.
